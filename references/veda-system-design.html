💡 What Veda Learn Does
Veda Learn is a passive AI mentor living inside VS Code. It watches your code silently via onDidChangeTextDocument. When you write a bug pattern, anti-pattern, security flaw, or performance smell, it teaches you why — not just fixes it.

Every lesson is delivered in three simultaneous channels: a natural Polly voice explanation, a Mermaid.js diagram, and a before/after code diff. A quiz fires 30 seconds after you acknowledge the lesson to confirm retention.
🎯 The 5 Detection Categories
Anti-Patterns — N+1 queries, God objects, mutable defaults

Bug Patterns — Null refs, unhandled promises, memory leaks

Security — SQL injection, hardcoded secrets, XSS vectors

Performance — O(n²) loops, blocking I/O, missing useMemo

Style/Convention — DRY violations, SOLID breaks, naming issues

⚡ Lambda Function Suite
  • 1
    auth.js
    Handles GitHub OAuth token exchange, upserts user into DynamoDB, issues signed JWT. Entry point for all identity.
  • 2
    analyze.js
    Receives code snapshot from extension, checks DynamoDB rate lock, calls Haiku 4.5 to classify, saves mistake, triggers lesson async.
  • 3
    lesson.js
    Runs 3 OpenRouter calls in parallel. Queries OpenSearch for RAG context. Calls Polly, uploads MP3 to S3, pushes lesson via WebSocket.
  • 4
    quiz.js
    Calls Haiku 4.5 to generate 2 MCQ questions as JSON for the concept that was just taught.
  • 5
    progressUpdate.js
    Calculates streak days, increments skill score, updates learning profile per language in DynamoDB.
  • 6
    wsConnect / wsDisconnect.js
    Store and remove connectionId in DynamoDB so lesson.js knows where to push the WebSocket message.
☁️ API Gateway: Two APIs
REST API — handles all synchronous operations. The extension calls it with a Bearer JWT for auth, analysis, quiz, and progress. Uses standard HTTP request/response.
WebSocket API — persistent bidirectional connection opened when the extension starts. lesson.js uses ApiGatewayManagementApi.postToConnection() to push lesson JSON to the correct VS Code client in real-time, without polling.

🔑 Auth Strategy: GitHub OAuth + Stateless JWT
  • 1
    No Password, No Email Sign-up
    GitHub OAuth is the single identity provider. Developers already have GitHub — zero friction to sign in.
  • 2
    JWT is Stateless
    No session store needed. Every Lambda verifies the JWT signature locally using JWT_SECRET. Payload: {userId, username, iat, exp}. Token expires in 30 days.
  • 3
    Secure Storage in VS Code
    context.secrets.store() uses the OS credential manager (Keychain on macOS, Credential Manager on Windows). Never in localStorage or settings.
  • 4
    URI Handler for Deep Link Return
    After OAuth, GitHub redirects to vscode://veda-learn.veda-learn/auth?token=JWT. VS Code's registered URI handler intercepts this and stores the token. No manual copy-paste.
  • 5
    WebSocket Auth via Query Param
    WebSocket connections cannot send headers. The JWT is passed as ?token=JWT in the WSS URL. wsConnect.js verifies it before storing the connectionId.
🛡️ Security Considerations
JWT Secret rotation: Stored in Lambda environment variables. Rotate by updating the env var — all old tokens instantly invalidated.

HTTPS only: API Gateway enforces TLS on all REST endpoints. No HTTP fallback.

CORS: REST API Gateway configured to allow only the VS Code webview origin. Prevents CSRF from browser-based attacks.

Rate limit per user: DynamoDB TTL items enforce 20 API calls/minute per userId. Prevents credit drain and abuse.

No GitHub token stored: The GitHub access_token is used once in auth.js to fetch user info, then discarded. Only the userId and JWT are persisted.
Auth Guard Middleware (every protected Lambda):
const token = event.headers?.Authorization?.replace('Bearer ', '');
const {userId} = jwt.verify(token, JWT_SECRET);

Returns 401 if missing, expired, or tampered.

📡 Real-Time Delivery: Why WebSocket over SSE
Server-Sent Events (SSE) require a persistent HTTP connection, which is incompatible with AWS Lambda's stateless, short-lived execution model.

API Gateway WebSocket API solves this cleanly. The VS Code extension holds one persistent WSS connection. When lesson.js finishes generating (after ~4–6 seconds of parallel AI calls + Polly), it calls ApiGatewayManagementApi.postToConnection(connectionId, payload) to push the lesson directly to the correct client.

The connectionId is looked up from veda-ws-connections DynamoDB table using the userId extracted from the JWT.
🧩 Parallel Lesson Generation
The three OpenRouter calls inside lesson.js run with Promise.all() — they execute simultaneously, not sequentially. This is critical for keeping lesson latency under 5 seconds:

Call 1 — Sonnet 4.5: Voice-optimized explanation (120 words, no markdown)
Call 2 — Sonnet 4.5: Before/after code fix as JSON
Call 3 — Gemini Flash: Mermaid.js diagram syntax

After the parallel calls resolve, RAG context is fetched from OpenSearch, then Polly synthesizes the audio. Total time: ~4–7 seconds from analyze trigger to WebSocket push.
🔄 Complete End-to-End Flow Summary
  • 1
    Developer types code in VS Code
    onDidChangeTextDocument fires on every keystroke. A 30-second debounce timer resets on each event. Only fires the API call after 30 seconds of no typing.
  • 2
    analyze.js classifies the code
    Haiku 4.5 receives the full file content, returns structured JSON with mistakeType, conceptId, severity, and confidence. Only proceeds if confidence ≥ 0.85.
  • 3
    Rate lock acquired, mistake saved
    A DynamoDB TTL item (30s) prevents duplicate lessons. The mistake is saved to veda-mistakes. lesson.js is invoked asynchronously — analyze.js returns 200 immediately.
  • 4
    Lesson generated in parallel
    3 OpenRouter calls run simultaneously. RAG context fetched from OpenSearch. Amazon Polly synthesizes MP3. MP3 uploaded to S3. Presigned URL generated (1hr validity).
  • 5
    Lesson pushed via WebSocket
    lesson.js calls ApiGatewayManagementApi with the connectionId from DynamoDB. VS Code extension receives the lesson JSON + audioUrl over the persistent WSS connection.
  • 6
    3-Panel sidebar renders
    Webview renders: Panel 1 (explanation with typewriter), Panel 2 (before/after diff with highlight.js), Panel 3 (Mermaid diagram). Polly MP3 auto-plays via presigned S3 URL.
  • 7
    Quiz fires after acknowledgment
    30 seconds after "Got it ✓" is clicked, quiz.js generates 2 MCQ questions via Haiku 4.5. Developer answers → confetti on correct → progressUpdate.js increments score and streak in DynamoDB.
